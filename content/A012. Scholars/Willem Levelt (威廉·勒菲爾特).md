# Willem Levelt (威廉·勒菲爾特)

#scholars #linguistics #cognitive_science

## Speaking: From Intention to Articulation (說話：從意圖到發音) (1989)

- **The Blueprint of the Speaker (說話者藍圖)**： 此概念為勒菲爾特言語產生模型的總體架構，描繪了一個從抽象意圖轉化為具體語音的完整過程。此藍圖將言語產生視為一個由多個獨立但相互協作的處理模組構成的系統。每個模組負責特定的任務，並以序列化的方式運作，意即前一階段的輸出成為後一階段的輸入。此模型的核心特徵是其高度模組化的設計，各個組成部分，如概念形成器、形式產生器與發音器官，各自擁有專門的知識庫與處理程序，且彼此間的資訊流動是單向的。例如，語法編碼的過程無法直接影響最初的溝通意圖。這種架構強調了言語產生的自動化與效率，因為各個模組可以平行處理其內部的子任務，同時整體流程保持清晰的階段性。此藍圖不僅是一個理論模型，它也為研究言語失誤、失語症以及言語發展提供了強有力的解釋框架，透過分析系統中特定模組的故障，可以推斷出言語障礙的根源。

- **Conceptualizer (概念形成器)**： 此為言語產生模型的第一個主要處理階段，其核心功能是生成說話者希望傳達的訊息內容。這個過程本身是非語言性的，它處理的是純粹的思想、意圖與概念。概念形成器從說話者的通用知識、情境知識以及對話歷史中提取相關資訊，並將其轉化為一個結構化的言前訊息（Preverbal Message）。此階段可細分為兩個子過程：宏觀規劃（Macroplanning）與微觀規劃（Microplanning）。宏觀規劃負責決定溝通的總體目標，例如是請求、告知還是說服，並選擇要表達的核心訊息。微觀規劃則負責為這些訊息賦予一個特定的視角或焦點，決定訊息的組織方式，例如將哪個元素設定為主題，哪個設定為評論。概念形成器的輸出是一個包含了所有必要語義與語用訊息的藍圖，但這個藍圖尚未披上任何語言的外衣，它是一個待語言化的純粹意念結構。

- **Macroplanning (宏觀規劃)**： 作為概念形成器中的首要步驟，宏觀規劃處理的是最高層次的溝e通意圖。它的任務是將一個模糊的、初始的溝通目標轉化為一系列具體的、待傳達的訊息片段。說話者在此階段會決定要說什麼的整體內容，但尚未決定如何用具體的詞句來表達。這涉及到從長期記憶中提取相關的知識，評估當前的溝通情境，並設定一個清晰的言談目標。例如，當一個人想向朋友借錢時，其宏觀規劃會包含幾個關鍵訊息：表達自己需要錢、說明需要錢的原因、提出借款的請求、以及可能提及還款的承諾。這些訊息片段在宏觀規劃階段被選定與排序，形成一個連貫的溝通策略。這個過程的成敗直接影響到後續言語內容的相關性與清晰度。宏觀規劃的輸出是一系列有序的言談意圖，它為下一階段的微觀規劃提供了內容基礎。

- **Microplanning (微觀規劃)**： 在宏觀規劃確立了要說什麼之後，微觀規劃接手處理如何組織這些訊息。此階段的主要任務是為言前訊息賦予一個特定的訊息結構（Information Structure），使其更適合進行語言編碼。這包括決定句子中的主題（topic）與焦點（focus），選擇適當的視角（perspective），以及安排訊息的呈現順序。例如，對於「貓追老鼠」這個事件，微觀規劃可以決定是以貓為主體（貓正在追老鼠），也可以是以老鼠為主體（老鼠正在被貓追）。這種選擇會影響後續語法編碼中主動語態與被動語態的使用。微觀規劃也負責添加語用學上的細節，例如確定語氣是肯定、疑問還是命令。此過程將宏觀規劃產生的抽象訊息片段，轉化為一個更為具體、帶有特定觀點與結構的言前訊息，使其準備好進入形式產生器進行詞彙與語法的編碼。

- **Preverbal Message (言前訊息)**： 這是概念形成器階段的最終產物，也是形式產生器階段的直接輸入。言前訊息是一個高度結構化但完全非語言性的訊息表徵。它包含了所有準備被語言化的語義與語用資訊，但其中不包含任何具體的詞彙或語法結構。可以將其想像成一張詳細的建築藍圖，上面標明了每個房間的功能、大小與相對位置，但沒有指定具體的建材或顏色。言前訊息指定了事件的參與者、動作、狀態以及它們之間的關係，同時也標記了訊息的焦點、主題以及說話者的態度。勒菲爾特的模型強調，言前訊息的產生是一個獨立於語言系統的過程，這意味著思想的形成先於語言的選擇。這個概念對於解釋跨語言現象至關重要，因為不同語言的使用者可能從相同的言前訊息出發，但透過各自語言的語法與詞彙規則，最終產生出結構迥異的句子。

- **Formulator (形式產生器)**： 此為言語產生模型的第二個主要處理階段，其核心任務是將來自概念形成器的抽象言前訊息轉化為一個具體的、可供發音的語音計畫（Phonetic Plan）。形式產生器是一個高度複雜的語言引擎，它內部包含了說話者所掌握語言的全部語法與詞彙知識。這個過程可以再細分為兩個主要的子系統：語法編碼（Grammatical Encoding）與語音編碼（Phonological Encoding）。語法編碼負責選擇適當的詞彙並將它們組合成合乎語法的句子結構。語音編碼則負責提取這些詞彙的語音形式，並將它們組織成一個包含音素、音節、重音和語調的詳細發音指令。形式產生器的運作是高度自動化且迅速的，說話者通常無法意識到其內部複雜的運算過程。它的效率與準確性直接決定了言語的流暢度與正確性。

- **Grammatical Encoding (語法編碼)**： 作為形式產生器的第一步，語法編碼負責將言前訊息翻譯成語法結構。這個過程始於詞彙選擇（Lexical Selection），系統根據言前訊息中的語義概念，在心理詞典（Mental Lexicon）中搜尋並啟動對應的詞條（Lemma）。一旦詞條被選定，其所附帶的句法框架（Syntactic Frame）便會被啟動，這個框架規定了該詞彙可以如何與其他詞彙組合。例如，動詞「給予」的詞條會指定它需要一個施事者主語、一個接受者間接賓語和一個被給予物直接賓語。接著，系統會根據這些句法框架，建構出一個完整的句法結構，即所謂的表層結構（Surface Structure），並將選定的詞條放入這個結構的相應位置。這個過程確保了最終產生的句子在語法上是正確的，並且準確地反映了言前訊息所要表達的語義關係。

- **Phonological Encoding (語音編碼)**： 在語法編碼完成之後，形式產生器進入語音編碼階段。此階段的任務是為已經排列在句法結構中的詞條檢索其具體的發音形式，即詞形（Lexeme），並將這些音韻訊息整合成一個可以由發音器官執行的語音計畫。這個過程從語法編碼產生的表層結構開始，系統逐一提取每個詞條對應的詞形，這些詞形包含了構成單詞的音素、音節結構以及重音模式等資訊。接著，系統會將這些孤立的詞形發音串連起來，並應用一系列音韻規則，例如連音、變調等，來生成一個流暢的、包含完整韻律訊息（如語調、節奏）的發音序列。這個最終的產物被稱為語音計畫（Phonetic Plan）或發音編譜（Articulatory Score），它是一個詳細到足以指導肌肉運動的指令集。語音編碼的順利進行是保證發音清晰、自然的關鍵。

- **Articulator (發音器官)**： 此為言語產生模型的第三個，也是最後一個主要處理階段。它的功能是接收來自形式產生器的語音計畫，並將其轉化為實際的、可被聽見的聲音。發音器官本身是一個生理系統，包括肺、喉、聲帶、舌頭、嘴唇、牙齒和顎等。在言語產生過程中，這個系統作為一個執行器，精確地執行語音計畫中編碼的每一個發音指令。肺部提供氣流，聲帶根據指令振動或不振動以產生濁音或清音，口腔和鼻腔的形狀則透過舌頭、嘴唇等部位的協同運動不斷改變，從而塑造出不同的元音和輔音。整個過程需要極其精細和快速的肌肉協調。發音器官的輸出就是我們實際聽到的外顯言語（Overt Speech）。這個階段的任何生理缺陷或運動控制障礙都會直接導致發音不清或其他言語障礙。

- **Mental Lexicon (心理詞典)**： 這是儲存在說話者大腦中的一個龐大的、高度組織化的詞彙知識庫。它不僅僅是一個單詞列表，而是一個複雜的網絡結構，包含了每個詞彙的多方面資訊。根據勒菲爾特的模型，心理詞典中的每個詞彙條目都至少包含兩個層次：詞條（Lemma）層次與詞形（Lexeme）層次。詞條層次儲存了詞彙的語義資訊（意義）和句法資訊（詞性、可搭配的句法框架等）。詞形層次則儲存了詞彙的音韻資訊（發音、音節結構、重音等）。這兩個層次的分離是勒菲爾特模型的一個核心假設，它能有效地解釋許多言語現象，例如「舌尖現象」（Tip-of-the-tongue state），即說話者知道自己想說哪個詞的意義和語法屬性（詞條層次已成功提取），但就是想不起它的具體發音（詞形層次的提取失敗）。

- **Lemma (詞條)**： 在勒菲爾特的模型中，詞條是心理詞典中代表一個詞彙的抽象語法單元。它不包含任何音韻資訊，而是純粹的語義與句法資訊的集合。當語法編碼器根據言前訊息選擇詞彙時，它首先提取的是詞條。一個詞條包含了該詞的核心意義以及其作為一個語法單位的所有規則。例如，動詞「eat」的詞條會包含其「攝取食物」的語義，同時也包含其作為一個及物動詞或不及物動詞的句法屬性。選擇了正確的詞條，就意味著說話者已經確定了要使用的詞彙的意義和它在句子中的語法角色。這個概念的提出，將詞彙提取過程分為了兩個階段：先確定意義和語法（選擇詞條），再確定發音（提取詞形），這極大地增加了模型的解釋力。

- **Lexeme (詞形)**： 詞形是心理詞典中與特定詞條相關聯的音韻形式表徵。它儲存了一個詞彙的具體發音所需的所有資訊，包括構成該詞的音素序列、音節劃分、重音位置等。在言語產生過程中，詞形的提取發生在詞條選擇之後的語音編碼階段。一旦一個詞條在語法編碼中被選定並置入句法框架，語音編碼器就會啟動並提取與該詞條相對應的詞形。例如，在選定了代表「行走」這個概念的詞條 `WALK` 之後，語音編碼器會提取其對應的詞形 /wɔːk/。如果語法上需要過去式，系統則會提取或生成其過去式詞形 /wɔːkt/。詞形的概念強調了意義/語法與聲音形式的分離，這種分離是理解言語產生過程中不同類型錯誤（如語義錯誤與語音錯誤）的關鍵。

- **Monitor / Speech Comprehension System (監控器 / 言語理解系統)**： 這是勒菲爾特模型中一個至關重要的部分，它是一個負責檢查與修正言語產出錯誤的認知機制。勒菲爾特提出，說話者擁有一個內建的言語理解系統，這個系統不僅能用來理解他人的話語，也能用來監控自己的言語產生過程。監控器可以透過兩條主要路徑運作：內在迴路（Internal Loop）與外在迴路（External Loop）。它不斷地將正在產生或已經產生的言語訊號與最初的溝通意圖進行比對。一旦檢測到不匹配或錯誤，監控器就會中斷當前的言語流，並啟動修復程序。這個機制的存在解釋了為何人類說話時能夠即時發現並修正自己的口誤、語法錯誤或用詞不當。監控器的效率直接影響到言語的準確性與流暢性。

- **Internal Loop (內在迴路)**： 這是監控系統的一條重要路徑，它允許說話者在言語實際發出之前，對其進行檢查。內在迴路監控的是形式產生器產生的「語音計畫」（Phonetic Plan），也就是所謂的「內在言語」（Inner Speech）。當語音計畫生成後，但在發音器官執行它之前，這個計畫會被傳送到言語理解系統進行預先審查。如果監控器在這個階段發現了錯誤，例如一個錯誤的音素或詞彙，它可以在不發出任何聲音的情況下進行修正。這就是所謂的「隱蔽修復」（covert repair）。內在迴路的存在解釋了為何有時說話者會突然停頓，然後用一個不同的詞或結構繼續說下去，聽者卻完全不知道之前曾有一個錯誤的計畫被中止了。這是一個極其快速且高效的品質控制機制。

- **External Loop (外在迴路)**： 這是監控系統的另一條路徑，它處理的是已經被發音器官產生出來的、實際可聞的外顯言語（Overt Speech）。說話者和聽眾一樣，會透過自己的聽覺系統接收到自己說出的話語。外在迴路就是利用這個聽覺反饋來檢查言語的正確性。當監控器透過這條路徑檢測到錯誤時（例如，說出了「我去了圖書館...呃，我的意思是銀行」），說話者會進行「公開修復」（overt repair）。這種修復通常伴隨著編輯性表達，如「呃」、「不對」、「我的意思是」等。外在迴路雖然比內在迴路慢，因為它需要等到聲音實際產生後才能介入，但它對於修正那些逃脫了內在迴路監控的錯誤，以及應對由發音器官本身造成的執行錯誤（如口齒不清）是必不可少的。

- **Information Encapsulation (資訊封裝)**： 這是勒菲爾特模型從認知科學家傑里·福多（Jerry Fodor）的模組化心智理論中借用的一個核心原則。它指的是言語產生系統中的各個處理模組（如概念形成器、形式產生器）是相對獨立運作的，各自擁有專屬的資訊與處理程序，並且不受其他模組的直接干擾。例如，形式產生器在進行語法編碼時，它只能接收來自概念形成器的言前訊息作為輸入，而無法反過來影響概念形成器去改變溝通意圖。同樣地，語音層面的知識（如某個音節是否容易發音）通常不會影響詞彙選擇（選擇哪個詞來表達意思）。這種封裝性確保了處理過程的高效與自動化，但也解釋了某些特定類型的言語失誤。例如，當一個詞的音韻形式（詞形）與另一個詞相似時，可能會發生音誤替換，即使這兩個詞的意義（詞條）相去甚遠，這正是因為語音編碼階段的處理是相對獨立於語義監控的。
