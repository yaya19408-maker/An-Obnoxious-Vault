# Alan Turing (艾倫·圖靈)

#scholars #computer_science #math

## On Computable Numbers, with an Application to the Entscheidungsproblem 論可計算數及其在判定問題上的應用 (1936)

- **Turing Machine (圖靈機)**： 一種抽象的數學模型，用以界定可計算性的邊界。此模型並非實體硬體的藍圖，而是一種思想實驗的工具，其構成包含一條無限長的紙帶、一個讀寫頭、一套有限的狀態以及一組轉移函數。紙帶被分割成連續的儲存格，每個儲存格可以儲存一個符號。讀寫頭能夠在紙帶上左右移動，讀取當前儲存格的符號，並根據內部狀態與讀取的符號，改寫儲存格的內容、改變自身狀態，以及決定下一步的移動方向。轉移函數是此機器的核心規則集，它精確地指示了在任何給定的狀態與符號組合下，機器應執行的操作序列。圖靈機的運作方式模擬了一位遵循固定規則進行計算的人類數學家，將直觀的演算法概念轉化為一個形式化、精確的系統。透過這個看似簡單的裝置，圖靈得以探索所有可能演算法的極限，並證明某些數學問題是無法透過任何演算法解決的。它的重要性在於提供了一個普適的計算框架，任何能夠被演算法描述的過程，原則上都可以由一台圖靈機來模擬執行。

- **Computable Numbers (可計算數)**： 指那些其小數或二進制表示能夠由一台圖靈機生成的實數。圖靈機從一個空白的紙帶開始，透過其預設的程式規則，能夠依序印出該數字的每一位數。這個概念將抽象的圖靈機與具體的數學對象聯繫起來。一個數字若為可計算的，意味著存在一個明確的、有限的程序來產生它的任意精確度的近似值。例如，圓周率π和自然常數e都是可計算數，因為存在演算法可以計算出它們小數點後的任何一位。圖靈透過這個定義，劃分了數學世界的兩類數字：一類是具有有限描述且可由機械程序產生的，另一類則是無法透過任何演算法捕捉的。這個概念揭示了並非所有數學上明確定義的數字都具有演算法上的可構造性。絕大多數的實數實際上是不可計算的，這顯示了可計算世界在整個實數連續體中所佔的比例極小，從而突顯了演算法能力的內在限制。

- **The Halting Problem (停機問題)**： 一個在計算理論中至關重要的判定問題。它提問：是否存在一個通用的演算法，能夠在輸入任意一個程式的描述和其初始輸入後，判斷該程式最終會停止執行，還是會陷入無限循環。圖靈證明了這樣一個通用的演算法是不存在的。他的證明採用了反證法：首先，假設存在這樣一個能夠解決停機問題的程式，稱之為Halt。接著，構造一個新的、有些矛盾的程式Trouble。Trouble程式會將另一個程式P的描述作為輸入，然後利用Halt來分析P在以自身為輸入時是否會停機。如果Halt的結論是P會停機，那麼Trouble就進入一個無限循環；反之，如果Halt的結論是P會無限循環，那麼Trouble就立刻停機。最後，將Trouble程式本身作為其自身的輸入來執行。此時，如果Trouble會停機，根據其定義，它應該進入無限循環；如果Trouble會無限循環，根據其定義，它又應該立刻停機。這種邏輯上的矛盾證明了最初的假設是錯誤的，也就是說，能夠解決停機問題的通用程式Halt不可能存在。這個結論意味著我們無法創造一個完美的程式碼偵錯工具，能夠預測所有程式的終極行為，揭示了計算本身固有的、不可逾越的限制。

- **Universal Turing Machine (通用圖靈機)**： 一種特殊的圖靈機，其設計使其能夠模擬任何其他圖靈機的行為。傳統的圖靈機是專用機器，其內部規則被寫死，只能執行單一的特定任務。通用圖靈機則不同，它將要模擬的機器的描述（即其轉移函數）以及該機器的初始輸入都寫在自己的紙帶上。通用圖靈機讀取這份描述，然後一步步地模擬目標機器的運作。它讀取目標機器的狀態和符號，查找描述中的對應規則，然後在紙帶的另一部分更新目標機器的模擬狀態和紙帶內容。這個概念是現代通用計算機的理論先驅。它首次在數學上形式化了這樣一種思想：一台機器可以執行任何可被描述的計算任務，只要給予它正確的程式。這正是 stored-program computer（儲存程式型電腦）的核心理念，程式本身和資料一樣，都可以被儲存在記憶體中並由處理器讀取和執行。通用圖靈機的存在證明了計算的普適性，為後來馮·諾伊曼架構等計算機設計提供了堅實的理論基礎。

- **Church-Turing Thesis (邱奇-圖靈論題)**： 這是一個關於有效計算（effective computability）本質的假說，它斷言任何在直觀意義上可被視為演算法性計算的函數，都可以由一台圖靈機來計算。同時，阿隆佐·邱奇也提出了等價的λ演算模型。此論題並非一個可以被數學證明的定理，因為直觀上的可計算性是一個非形式化的概念。然而，它匯集了來自不同數學家提出的多種獨立的計算模型，包括圖靈機、λ演算、遞歸函數等，而這些模型後來都被證明在計算能力上是等價的。這種等價性為論題提供了強而有力的支持。邱奇-圖靈論題是計算機科學的哲學基石，它為演算法這個概念提供了一個穩定且公認的形式化定義。它意味著，如果一個問題無法被圖靈機解決，那麼它也無法被任何其他可以想像的、遵循確定性步驟的計算設備所解決。這為探索不可計算問題的領域劃定了清晰的界線。

- **Entscheidungsproblem (判定問題)**： 由德國數學家大衛·希爾伯特在1928年提出的挑戰，是其宏大計畫中的一個核心問題。判定問題尋求一個通用的演算法，該演算法能夠接收任何一個形式化的一階邏輯語句作為輸入，並在有限時間內判斷該語句是普遍有效的（即在所有可能的解釋下都為真）還是無效的。希爾伯特期望找到這樣一個機械化的程序來一勞永逸地解決所有數學證明問題。圖靈在其1936年的論文中，透過將判定問題與圖靈機的停機問題聯繫起來，給出了否定的答案。他證明了停機問題是不可判定的，接著他展示了如何將任何一個圖靈機的停機問題實例轉化為一個等價的一階邏輯語句。如果判定問題有解，那麼這個解也能夠被用來解決停機問題。既然停機問題無解，那麼判定問題也必定無解。這個結論對數學基礎產生了深遠的影響，它粉碎了希爾伯特關於數學形式系統完備性與可判定性的樂觀信念，揭示了數學真理的世界中存在著無法透過純粹機械化推理來觸及的領域。

## Bletchley Park and Cryptanalysis 布萊切利園與密碼分析 (c. 1939-1945)

- **Banburismus (班伯里方法)**： 一種由圖靈在布萊切利園開發的統計密碼分析技術，主要用於破解德國海軍使用的恩尼格瑪密碼機。與陸軍和空軍不同，海軍恩尼格瑪機具有更高的複雜性，使得純粹的蠻力攻擊變得不切實際。班伯里方法的核心思想是利用貝氏統計來推斷恩尼格瑪機轉子的初始設置。此方法不需要完整的明文猜測，而是基於一個假設：兩段不同的密文可能源於相同的明文片段（例如，標準的天氣報告）。分析人員會將兩段密文並排對齊，逐個字符比較。如果這兩段密文確實由相同的明文加密而來，那麼它們之間的某些統計特性會顯現出來。圖靈設計了一套計分系統，用以量化支持或反對轉子設置假設的證據強度。透過在大量穿孔卡片（稱為Banburys）上進行系統性的比較和計分，密碼分析員可以逐步排除不可能的轉子組合，從而將炸彈機需要搜索的範圍縮小到一個可管理的規模。這種方法體現了圖靈將抽象數學和統計學原理應用於實際情報工作的能力。

- **The Bombe (炸彈機)**： 一種為了加速破解恩尼格瑪密碼而設計的機電設備。雖然其概念源自波蘭密碼局的Bomba，但圖靈的設計在邏輯上更為通用和強大。炸彈機的任務是尋找恩尼格瑪機當天的轉子順序和初始位置。其工作原理基於對已知或猜測的明文片段（稱為crib）的利用。當分析員有了一段可能的明文和其對應的密文時，他們就建立了一個描述明文、密文和恩尼格瑪機內部電路之間關係的邏輯鏈。炸彈機透過模擬多組恩尼格瑪機的轉子組合來並行測試這些邏輯鏈。它會快速遍歷數萬種可能的設置，尋找不會產生邏輯矛盾的組合。當機器找到一個可能的設置時，它會停下來，操作員再手動驗證這個設置是否正確。圖靈的主要貢獻在於其邏輯設計，他指明了機器需要如何利用密文中的循環結構來同時測試多個假設並快速排除錯誤路徑，而機器的工程實現則由哈羅德·基恩完成。

- **Turingery (圖靈方法)**： 這是布萊切利園內部對圖靈為破解洛倫茲密碼（德軍高層使用的電傳打字機密碼，代號為Tunny）所開發的統計分析技術的暱稱。洛倫茲密碼比恩尼格瑪更為複雜，其加密過程涉及十二個轉輪。圖靈意識到，直接的蠻力攻擊是不可行的。他的方法是分析密文本身的統計特性，而不需要猜測明文。他發現，透過對密文流與其自身進行特定偏移量的異或操作，可以消除明文的影響，從而暴露出加密所用偽隨機序列的統計偏差。這種差分分析技術使得分析員能夠推斷出洛倫茲機其中兩個轉輪的設定模式。這個過程最初是手動完成的，極其繁瑣且耗時。圖靈的這套方法學為後續的自動化破解奠定了基礎，最終促成了世界上第一台可程式化的電子計算機巨人計算機（Colossus）的誕生，巨人計算機的任務就是自動化執行圖靈所設計的統計計算。

## Post-War Computer Architecture 戰後計算機架構 (c. 1945-1947)

- **Automatic Computing Engine (ACE) (自動計算引擎)**： 這是圖靈在二戰後於英國國家物理實驗室提出的第一代儲存程式型電子計算機的詳細設計方案。ACE的設計理念與當時在美國發展的EDVAC等計算機有所不同，它體現了圖靈對效率和速度的極致追求。其核心特點包括一個極簡但功能強大的指令集、對分層記憶體的早期構想以及對子程式概念的重視。圖靈的設計採用了水銀延遲線作為高速主記憶體，這種記憶體以聲波脈衝的形式儲存數據。為了最大化計算效率，ACE的指令設計允許程式設計師精確控制下一條指令在記憶體中的儲存位置，從而減少因等待數據從延遲線中循環出來而造成的延遲。這種非線性的程式設計方式雖然複雜，但能實現極高的運算速度。圖靈還提出了一套縮寫計算機指令（Abbreviated Computer Instructions），這可以被視為一種早期的組合語言或程式設計語言，旨在簡化程式的編寫過程。儘管完整的ACE從未按照圖靈的原始宏大設計建造，但其簡化版本Pilot ACE於1950年成功運行，並對英國早期的計算機發展產生了重要影響。

- **Hierarchical Memory (分層記憶體)**： 在ACE的設計報告中，圖靈明確地闡述了不同速度和容量的儲存設備協同工作的思想，這正是現代計算機分層記憶體架構的雛形。他認識到，最快的記憶體（水銀延遲線）既昂貴容量又小，而較慢的儲存媒體（如磁鼓）則可以提供更大的容量和更低的成本。因此，他的設計構想是將程式和數據在這些不同層級的記憶體之間進行轉移。頻繁使用的指令和數據應存放在高速記憶體中，而不常用的則存放在較慢的大容量儲存中。這個概念預見了現代計算機系統中快取（Cache）、主記憶體（RAM）和輔助儲存（硬碟或固態硬碟）之間的關係。透過智慧地管理數據在不同層級間的流動，可以在成本可控的範圍內，實現接近最快記憶體速度的整體系統性能。這種對成本與性能之間權衡的深刻理解，展現了圖靈不僅是數學家，也是一位具有遠見的計算機架構師。

- **Subroutine Library (子程式庫)**： 圖靈在規劃ACE的應用時，不僅僅考慮了硬體設計，也深入思考了軟體層面的問題。他提出了一個建立標準化子程式庫的構想。這個庫將包含一系列經過最佳化和驗證的常用數學運算程式，例如計算對數、三角函數、平方根以及執行矩陣運算等。其他程式設計師在編寫自己的應用程式時，可以直接調用這些預先寫好的子程式，而無需重新發明輪子。這個概念是模組化程式設計和軟體重用的基石。它不僅能極大地提高程式開發的效率和可靠性，也促進了程式設計知識的積累和共享。圖靈的願景是建立一個豐富的軟體生態系統，讓計算機不僅僅是一台能執行指令的機器，更是一個擁有強大軟體工具支持的解決問題的平台。這個想法在今天已經成為所有程式設計領域的標準實踐。

## Computing Machinery and Intelligence 計算機器與智慧 (1950)

- **The Imitation Game / Turing Test (模仿遊戲 / 圖靈測試)**： 這是圖靈為迴避對機器是否能思考這一哲學問題的直接定義而設計的一個操作性測試。在其1950年的論文中，他提出了一個名為模仿遊戲的場景。遊戲中有三位參與者：一位人類提問者C，一位人類男性A，以及一位人類女性B。提問者C與A和B被物理隔離，只能透過文字界面進行交流。C的目標是判斷哪一位是男性，哪一位是女性。在這個基礎遊戲中，A的任務是誤導提問者，而B的任務是幫助提問者。圖靈接著提出，如果我們將參與者A替換成一台機器，而提問者的任務變成判斷哪個是人類，哪個是機器。如果機器能夠在遊戲中成功地模仿人類，以至於提問者判斷錯誤的機率與其在區分男性和女性時相當，那麼我們就可以認為這台機器具有智能。圖靈測試的巧妙之處在於，它將智慧的定義從內在的、無法觀測的心理狀態，轉化為外在的、可觀測的行為表現。它不問機器是否真的在思考，只問它是否能表現出與人類無法區分的思考行為。

- **Arguments from Various Disabilities (源於各種無能的論點)**： 在其關於機器智慧的論文中，圖靈預見並系統性地回應了九種反對機器能夠思考的常見論點。他將這些論點歸類為源於各種無能的論點，意指這些反對意見都聲稱機器永遠無法做到某件特定的事情。這些論點包括：神學論點（思考是上帝賦予人類不朽靈魂的功能）；畏首畏尾論點（機器思考的後果太可怕，所以最好不要相信它可能發生）；數學論點（哥德爾不完備定理等邏輯結果限制了任何離散狀態機器的能力）；意識論點（除非機器能感受情緒、能寫十四行詩，否則不能說它在思考）；原創性論點（機器只能做我們程式設定它做的事，無法創造新事物）；以及非形式行為論點（人類行為無法被任何有限的規則集所捕捉）。對於每一個論點，圖靈都給出了精闢的反駁。例如，針對意識論點，他指出這實際上要求我們成為那台機器才能確定它是否在思考，這會導致唯我論的困境。針對原創性論點，他指出我們也常常對人類的創新感到驚訝，而機器的學習能力可能使其行為超越其初始程式的簡單範疇。

## The Chemical Basis of Morphogenesis 形態發生的化學基礎 (1952)

- **Reaction-Diffusion System (反應-擴散系統)**： 這是圖靈在晚年轉向數理生物學領域時提出的一個數學模型，用以解釋生物體形態（如動物斑紋、植物葉序）是如何從一個原本均勻的細胞群體中自發產生的。該系統由一組偏微分方程描述，模擬了兩種或多種被稱為形態發生素的化學物質在一個空間區域內的行為。系統包含兩個核心過程：反應（Reaction）和擴散（Diffusion）。反應過程描述了這些化學物質在局部如何相互作用，例如一種物質（活化劑）能夠催化自身和另一種物質（抑制劑）的產生；而另一種物質（抑制劑）則會抑制活化劑的生成。擴散過程描述了這些物質如何在組織中傳播開來，通常假設抑制劑的擴散速度遠快於活化劑。這個模型的關鍵在於，即使初始狀態是完全均勻的，活化劑和抑制劑之間不平衡的相互作用和擴散速率差異也會導致系統失穩，從而自發地形成穩定的、非均勻的空間圖案。

- **Turing Pattern (圖靈圖案)**： 指在反應-擴散系統中，由對稱性破缺自發形成的各種穩定空間圖案。當系統參數（如反應速率、擴散係數）處於特定範圍時，一個微小的隨機擾動會被放大，導致活化劑在某些區域聚集，形成峰值；同時，快速擴散的抑制劑會在這些峰值周圍形成抑制區域，阻止其他峰值的形成。這種局部活化與遠程抑制的機制最終會產生規則的、重複的圖案，如點狀、條紋狀或更複雜的迷宮狀結構。圖靈提出，這些抽象的數學圖案可能是自然界中許多生物形態的基礎，例如豹的斑點、斑馬的條紋、熱帶魚身上的花紋等。他用數學方程證明了，簡單的化學和物理定律足以從無序中創造出複雜的生物結構，而無需為每個細節預設一個藍圖。這個理論為發育生物學提供了一個全新的、基於自組織原理的視角。

- **Morphogen (形態發生素)**： 在圖靈的反應-擴散模型中，形態發生素是假設存在的化學信號物質，它們的濃度分佈決定了細胞的分化和組織的發育模式。圖靈模型通常涉及至少兩種形態發生素：一種是自催化的活化劑（Activator），它能促進自身產生，並在局部聚集；另一種是抑制劑（Inhibitor），它由活化劑觸發產生，但作用是抑制活化劑的活性。模型的關鍵在於兩種物質的擴散速率不同，通常抑制劑擴散得比活化劑快得多。當一個地方的活化劑濃度因隨機波動略微升高時，它會產生更多的活化劑和抑制劑。活化劑由於擴散慢而停留在原地，形成一個活化中心。而抑制劑則迅速擴散到周圍區域，壓制了附近其他活化中心的形成。這種機制確保了圖案的間距和規律性。圖靈的理論在提出時是純粹的數學推導，但在數十年後，生物學家們確實發現了在生物體內扮演類似角色的真實分子，驗證了形態發生素作為發育模式決定者的核心思想。
