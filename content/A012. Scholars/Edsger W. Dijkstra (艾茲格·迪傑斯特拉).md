# Edsger W. Dijkstra (艾茲格·迪傑斯特拉)

#scholars #computer_science #programming

## Early Work on Algorithms and Concurrency (1950s-1960s)

- **Dijkstra's Algorithm (迪傑斯特拉演算法)**： 一種用於在圖中尋找單一源頭最短路徑的演算法，適用於邊權重為非負數的圖。此演算法由迪傑斯特拉於1956年構思，並於1959年發表。其運作方式是從一個指定的起始節點開始，逐步擴展至相鄰節點，並持續更新從起始節點到各個節點的最短路徑估計值。演算法維護一個已訪問節點的集合和一個未訪問節點的集合。在每一步中，它會從未訪問節點中選出一個距離起始節點最近的節點，將其加入已訪問集合，然後更新所有與該節點相鄰的未訪問節點的路徑長度。如果通過新訪問的節點可以找到一條更短的路徑到達某個鄰居節點，則更新該鄰居節點的距離。這個過程重複進行，直到所有可達節點都被訪問過，或找到目標節點為止。為了有效率地選出距離最近的未訪問節點，此演算法通常會使用優先佇列（priority queue）作為其核心資料結構。迪傑斯特拉演算法是許多路由協定（如OSPF）和地理資訊系統（GIS）中路徑規劃功能的基礎。

- **Shunting-yard Algorithm (調車場演算法)**： 一種用於解析中序表示法（infix notation）數學運算式的方法，由迪傑斯特拉發明。其名稱源於其運作方式類似於鐵路調車場對車廂進行排序。此演算法的主要功能是將人類習慣閱讀的中序運算式（如 `3 + 4 * 2`）轉換為電腦更容易處理的後序表示法（postfix notation，或稱逆波蘭表示法 `3 4 2 * +`）或抽象語法樹（AST）。演算法使用一個佇列來存放最終的輸出，以及一個堆疊來暫存運算子。它逐一讀取中序運算式的符號（token）。如果是數字，則直接放入輸出佇列；如果是運算子，則將其與堆疊頂端的運算子進行優先級比較。若當前運算子的優先級高於堆疊頂端運算子，或堆疊為空，則將其推入堆疊。否則，將堆疊頂端的運算子彈出至輸出佇列，再重複比較，直到滿足條件為止。括號則用來改變優先級，左括號直接推入堆疊，右括號則觸發堆疊內的運算子持續彈出至輸出佇列，直到遇到左括號為止。此演算法是許多編譯器和計算機中語法解析器的重要組成部分。

- **THE Multiprogramming System (THE多道程式系統)**： 於1968年由迪傑斯特拉及其團隊在荷蘭埃因霍溫理工大學開發的一套具開創性的作業系統。THE系統的設計目標是為了證明，透過嚴謹的結構化設計，可以建構出正確且可靠的複雜軟體系統。其最顯著的貢獻是引入了分層式架構（Layered Architecture）。整個系統被劃分為六個層次，從最底層的第0層（處理器分配與時鐘）到最上層的第5層（使用者程式）。每一層只能使用其下一層提供的服務與功能，而對其他層的內部實作細節一無所知。這種嚴格的層級依賴關係極大地簡化了系統的設計、除錯與驗證。開發團隊能夠對每一層進行獨立的測試與形式化證明，確保其正確性，進而保證整個系統的穩定性。THE系統也首次應用了號誌（Semaphore）機制來處理行程間的同步問題，有效地解決了多道程式環境下的資源競爭與死結問題。它成為後續作業系統設計的重要典範，其設計原則影響深遠。

- **Layered Architecture (分層架構)**： 一種軟體設計模式，將系統組織成一系列堆疊的層次。每一層都為其上一層提供特定的服務，同時也作為其下一層的客戶端。此架構的核心原則是關注點分離（Separation of Concerns），每一層都專注於處理系統的某個特定面向。一個關鍵的規則是，某一層只能與其直接相鄰的下層進行互動，而不能跨層呼叫。這種單向依賴性降低了層與層之間的耦合度，使得系統更加模組化。修改某一層的內部實作，只要其提供的介面不變，就不會影響到其他層。這大大提高了系統的可維護性、可擴展性和可重用性。迪傑斯特拉在THE作業系統中的成功實踐，展示了分層架構在管理複雜系統方面的強大能力。此模式後來被廣泛應用於各種領域，例如網路通訊的OSI模型和TCP/IP協定堆疊，以及現代應用程式的資料存取層、業務邏輯層與表示層。

- **Semaphore (號誌)**： 由迪傑斯特拉提出的一種用於管理多個行程對共享資源存取的同步原語（Synchronization Primitive）。號誌本質上是一個受保護的整數變數，只能透過兩個原子操作來進行存取：P操作（來自荷蘭語Proberen，意為測試）和V操作（來自荷蘭語Verhogen，意為增加）。P操作（也稱為wait或down）會檢查號誌的值。如果值大於零，則將其減一並繼續執行。如果值為零，則行程會被阻塞，直到號誌的值變為正數。V操作（也稱為signal或up）會將號誌的值加一。如果有行程因P操作而被阻塞，V操作會喚醒其中一個行程。透過使用號誌，程式設計師可以有效地解決並行程式設計中的臨界區問題（critical section problem），確保在任何時候只有一個行程能夠存取特定的共享資源，從而避免資料競爭（race condition）和不一致性。號誌分為二進位號誌（值只能是0或1）和計數號誌（值可以為任意非負整數），前者用於實現互斥鎖（mutex），後者用於管理多個相同類型的資源。

- **Mutual Exclusion (互斥)**： 並行計算中的一個基本屬性，旨在防止多個行程同時進入其臨界區（critical section）。臨界區是指一段存取共享資源（如共享變數、檔案或資料庫）的程式碼。如果沒有互斥機制，多個行程同時修改共享資源可能會導致非預期的結果和資料損毀，這種情況稱為資料競爭。實現互斥是設計正確並行演算法的核心挑戰。迪傑斯特拉對此領域有重大貢獻，他不僅提出了第一個針對多處理器環境的純軟體互斥演算法（Dijkstra's algorithm for mutual exclusion），還發明了號誌作為一種更高級、更易於使用的同步工具。一個有效的互斥解決方案必須滿足三個條件：確保互斥性（任何時候最多只有一個行程在臨界區內）、避免死結（系統中的行程不會永遠等待無法獲得的資源）、以及避免餓死（任何請求進入臨界區的行程最終都能夠進入）。

- **Dining Philosophers Problem (哲學家就餐問題)**： 由迪傑斯特拉在1965年提出的一個經典同步問題，用以闡述在並行計算中可能出現的死結（Deadlock）和餓死（Starvation）問題。問題描述了五位哲學家圍坐在一張圓桌旁，他們交替地進行思考和用餐。每兩位哲學家之間放著一根叉子，總共有五根叉子。每位哲學家需要同時拿起左右兩手的叉子才能用餐。問題的挑戰在於設計一套規則，讓哲學家們可以順利地用餐，而不會導致系統停頓。一個簡單的策略是，每位哲學家先拿起左邊的叉子，再嘗試拿起右邊的叉子。然而，如果所有哲學家同時拿起左邊的叉子，他們都會發現右邊的叉子已被鄰居拿起，於是所有人都會永遠等待下去，這就形成了死結。此問題成為評估和比較不同並行程式設計同步機制（如號誌、監視器）有效性的標準模型。解決方案通常涉及破壞死結的四個必要條件之一，例如引入順序限制（要求哲學家按特定順序拿叉子）或使用一個仲裁者來協調資源分配。
